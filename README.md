## 📊 Структура базы данных и описание справочников

### Справочник 1: `airports` (Аэропорты) — Главный справочник

| Поле | Тип PostgreSQL | Описание |
| :--- | :--- | :--- |
| **id** | `SERIAL PRIMARY KEY` | Уникальный идентификатор (скрыт от пользователя) |
| **iata_code** | `VARCHAR(3) NOT NULL UNIQUE` | Международный 3-буквенный код IATA (MSQ, SVO, WAW) |
| **name** | `TEXT NOT NULL` | Полное название/описание аэропорта |
| **city** | `VARCHAR(100) NOT NULL` | Город расположения |
| **country** | `VARCHAR(100) NOT NULL` | Страна |
| **opened_date** | `DATE NOT NULL` | Дата открытия аэропорта |
| **runways_count** | `INTEGER NOT NULL` | Количество взлётно-посадочных полос (> 0) |
| **runway_length_km**| `NUMERIC(5, 2) NOT NULL` | Длина ВПП в километрах (> 0) |
| **altitude_m** | `INTEGER NOT NULL DEFAULT 0` | Высота над уровнем моря в метрах |

### Справочник 2: `flights` (Авиарейсы) — Зависимый справочник

| Поле | Тип PostgreSQL | Описание |
| :--- | :--- | :--- |
| **id** | `SERIAL PRIMARY KEY` | Уникальный идентификатор (скрыт от пользователя) |
| **flight_number** | `VARCHAR(10) NOT NULL UNIQUE`| Номер авиарейса (BT101, SU270, LO043) |
| **airport_from_id** | `INTEGER NOT NULL` | Аэропорт вылета (Внешний ключ `FK` к `airports.id`) |
| **airport_to_id** | `INTEGER NOT NULL` | Аэропорт прилёта (Внешний ключ `FK` к `airports.id`) |
| **departure_date** | `DATE NOT NULL` | Дата вылета |
| **duration_minutes**| `INTEGER NOT NULL` | Длительность полёта в минутах (> 0) |
| **distance_km** | `NUMERIC(7, 2) NOT NULL` | Расстояние маршрута в километрах (> 0) |
| **base_price_usd** | `NUMERIC(10, 2) NOT NULL`| Базовая цена билета в USD (>= 0) |
| **seats_total** | `INTEGER NOT NULL` | Общее количество пассажирских мест (> 0) |

## Описание связей и зависимостей между таблицами

1. **Связь «Один ко многим» (`1:M`)**
   Один аэропорт из таблицы `airports` может быть связан с множеством рейсов из таблицы `flights` (как в качестве точки вылета, так и в качестве точки прилёта).
2. **Ограничения целостности (Внешние ключи):**
   * Поля `airport_from_id` и `airport_to_id` ссылаются на первичный ключ `id` родительской таблицы `airports`.
3. **Поведение при обновлении (`ON UPDATE CASCADE`):**
   При изменении системного идентификатора аэропорта в главной таблице, изменения автоматически распространяются на все связанные записи в таблице рейсов.
4. **Поведение при удалении (`ON DELETE RESTRICT`):**
   **Защита данных:** Если в справочнике `flights` существует хотя бы один рейс, использующий определенный аэропорт, СУБД заблокирует попытку удаления этого аэропорта из таблицы `airports`. Это гарантирует, что данные зависимого справочника не будут повреждены или удалены случайно.
5. **Дополнительное ограничение (`CHECK`):**
   Аэропорт вылета и аэропорт прилёта не могут совпадать (`airport_from_id <> airport_to_id`).
